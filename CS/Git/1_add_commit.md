# 🚀 [TIL] 쉽게 정리하는 Git 기초 01. add와 commit 이해하기

## 1. Git이란 뭘까?
내가 짠 코드들을 기록하고 보관해주는 고마운 **'버전 관리 시스템'**이다. 
단순히 저장만 하는 게 아니라, 코드의 역사를 관리한다고 보면 된다.

- **과거로 타임머신 타기:** 코드를 짜다가 망쳤을 때, 언제든 멀쩡했던 이전 시점으로 되돌릴 수 있다.
- **히스토리 추적:** 내가 과거에 어떤 삽질을 했는지, 코드를 왜 이렇게 고쳤는지 기록을 다 볼 수 있다.
- **안전장치:** 실수로 파일을 날려도 Git에만 잘 기록해뒀다면 안심이다.

## 2. Git 설치와 초기 설정
### 1) 설치할 때 기억할 것
1. 구글에 **"git windows"** 검색해서 공식 사이트에서 설치 파일을 받았다.
2. 설치 옵션이 많은데, 나는 아래 두 가지는 꼭 챙기기로 했다:
    - 에디터를 **Visual Studio Code**로 설정하기
    - 기본 브랜치 이름을 **main**으로 고정하기 (나중에 헷갈리지 않게!)

### 2) 내 정보 등록하기 (최초 1회)
Git이 "이 코드는 누가 짠 거야?"라고 물어볼 때 답할 수 있도록 내 정보를 등록했다. 
1. 폴더에서 **Git Bash**를 열고 아래 명령어를 입력했다.
2. 내 이름과 이메일을 정확히 적어주는 게 포인트!
    ```bash
    git config --global user.name "Lee YeongHyeon"
    git config --global user.email "본인이메일@email.com"
    ```

## 3. Git 작업 흐름 정리 (Workflow)

이제 본격적으로 코드를 기록하는 순서를 정리해본다. 크게 3단계다.

### 1단계: 감시 시작 (`init`)
"이제부터 이 폴더의 변화를 지켜봐 줘!"라고 명령하는 것이다.
```bash
git init # 실행하면 숨겨진 .git 폴더가 생기면서 Git이 감시를 시작한다.
```

### 2단계: 장바구니에 담기 (`add`)
코드를 수정했다고 바로 저장되는 게 아니다. 저장하고 싶은 파일만 골라서 **'장바구니(Staging Area)'**에 먼저 담아야 한다.
```bash
git add text.txt # 이 파일만 담고 싶을 때
git add .        # 수정한 모든 파일을 한꺼번에 담고 싶을 때 (보통 이걸 많이 쓴다)
```

### 3단계: 기록 확정 (`commit`)
장바구니에 담긴 파일들을 실제로 **기록 보관소**에 집어넣는 단계다. 
```bash
git commit -m "어떤 작업을 했는지 한 줄 요약" 
# 나중에 내가 봤을 때 "아, 이때 이걸 고쳤지!"라고 알 수 있게 정성껏 적자.
```

## 4. 내 머릿속에 저장한 Git 동작 원리

파일이 최종 저장소에 가기까지의 과정을 그림으로 그려보니 더 이해가 잘 된다.

```mermaid
flowchart LR
    A[Working Directory<br/>내가 코드 짜는 곳] -- "git add" --> B[Staging Area<br/>장바구니 / 대기소]
    B -- "git commit" --> C[Local Repository<br/>로컬 저장소 / 기록 보관소]

    style A fill:#f9f,stroke:#333,stroke-width:2px,color:black
    style B fill:#bbf,stroke:#333,stroke-width:2px,color:black
    style C fill:#bfb,stroke:#333,stroke-width:2px,color:black
```

- **Working Directory:** 지금 내가 VS Code 켜놓고 열나게 코딩하고 있는 실제 현장이다.
- **Staging Area:** 커밋하기 전, "이번에 이거 이거 저장할 거야"라고 찜해둔 파일들이 잠시 머무는 곳이다.
- **Local Repository:** `commit` 명령을 내리면 파일들이 안전하게 박제되는 내 컴퓨터 안의 보물창고다.

## 📚 앞으로 더 공부해야 할 것들 (Roadmap)

기초를 뗐으니 이제 다음 단계로 넘어가 보자.

### Step 2. 과거 여행 떠나기 (`log`, `reset`)
- 내가 남긴 기록들을 훑어보고(`log`), 실수했을 때 시간을 되돌리는 법(`reset`) 공부하기.

### Step 3. 내 코드를 세상 밖으로 (`GitHub 연동`)
- 내 컴퓨터에만 있는 기록을 온라인 저장소인 깃허브에 올리는 법 배우기.

### Step 4. 평행우주 만들기 (`branch`)
- 원본은 놔두고 새로운 기능을 따로 만들어보는 가지치기 기술 익히기.

### Step 5. 같이의 가치 (`협업과 충돌`)
- 팀원과 코드를 합치고(PR), 의견 충돌이 났을 때 해결하는 법 마스터하기.
